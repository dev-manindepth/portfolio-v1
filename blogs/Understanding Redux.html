What is Redux

Redux is state management library for Javascript app. It is a library for managing and updating the application state using events called actions. It's a centralized store that's shared across the entire application and ensures that the state gets updated in a predictable fashion

The three main principles of Redux

  1. Single Source of Truth
  2. State is Immutable
  3. Changes are made with pure functions only

Let's understand all the core concept in details

Single Source of Truth means the global state of our application is stored in an object within a single store.

How does Redux Work

The working of Redux depends upon 3 core concepts

  1. Action
  2. Dispatch
  3. Reducer

Action

Action is a plan javascript object which describes what appropriate action need to be taken upon particular event occurance .
For example : When we click a button , clicking a button is event , but what we want to do on clicking a button is action sometimes we want to increment a counter ,sometimes we want to fetch list of users etc .

Let's reiterate what we have learnt . Action is a plain javascript object which describe the event . The action object must have a named key called type . We can also provide more information about the event using another name key payload . It is optional to define but very helpul at the same . for example -> On clicking a button you want to fetch list of users , but how many users Infinty ? .No probably no we don't want that much user data we probably want only 20 users . The extra information here 20 user can be passed as payload 


        const action ={ 
          type:"increment"
         } 
        const action ={
           type:"increment",
            payload:2 
        }
        const action ={ 
          type:"fetchUsers",
           payload:20
       }

Action Creator

That means Action Creators are function which returns action object . Now from now insted of writting plain action object we will create a action creator function which in turn return our action object

Dispatch

Now we have already defined a action , Now we need some way to send this action to store . Redux provides us a dispatch() function which carry our action .For ex - You want to buy ticket from railway station .You have your action prepared const action ={ type="buyTicket" } . and to send this action to ticket counter we need some way .For this we fill a form that describes our action and payload and give it to ticket counter

Redux store provides us dipatch() method . So first import store and then call dispatch() method on it 

        import store from './store.js' 
                 
        store.dispatch({type:"buyTicket"});

Reducers

Reducers are pure function which are responsible for all the state transition in store . In simple word reducers perform all the logic of our state . It is responsibe for making state transition. For ex -> Up until now We have our action ready and also we have dispatched that action through dispatch function . now we need some who can listen to our action . Reducer is that function which listend to any action dispatched through dispatch() function . In real world example A shopkepper can be reducer because It listens to all the action dispatched by us and also he is responsibe for selling returning or restocking item in store . A ticket master can be reducer because again he listens our action and perform some work like making reservation seat for us and reduce the count of seat by 1 etc. 

      const reducer = (state,action)=>{
          switch(action.type){
              case "buyTicket":
                return {
                  ...state,
       state.numOfTicket : 
       state.numOfTicket - action.payload
                  }
                  break;
              case "returnTicket":
                  return {
                    ...state,
        state.numOfTicket:
        state.numOfTicket + action.payload 
                  }
                  break;
              default:
                  return state;    
                
            }
        }

reducer fuction recieve two argument state (i.e previous state) of application action (it recieves the actions that we dispatch )
We defined a switch case inside reducer fuction it will check each until a matching case found . If any case matches then particular state transition occur . One thing to not here we donot try to mutate the original store state . Store is immutable . Hence we first copy all the object properties and only update the property that we want
But if the action sent by user doesn't match in switch case then we return the default previous state , This situation is similar to the situation where you want to buy something from a shop but the shop doesnot have that item .then the shopkepper
